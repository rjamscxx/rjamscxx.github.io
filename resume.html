How I Work in a Team

  
01
Understand before building
I read the spec, trace the codebase, and ask targeted questions before writing a line. I never assume — I verify.

  
02
Branch, build, test
Feature branch off main, build the smallest working increment, run E2E tests. Commit atomically with descriptive messages that make sense in git log.

  
03
Open a clean PR
PR description explains what changed, why, and how to test it. Screenshots for UI changes. No unexplained diffs.

  
04
Iterate on feedback
Review comments are opportunities, not criticisms. I address each one, explain my reasoning when I disagree, and merge only when the team is satisfied.
